# koishi-plugin-chatluna-music-voice

为 ChatLuna 提供音乐搜索工具，让 AI 可以搜索网易云音乐或 QQ 音乐，并按配置发送语音、链接、文件或音乐卡片

## 功能

- 支持网易云音乐搜索
- 支持 QQ 音乐搜索，可与网易云音乐混合搜索
- 支持每个平台各返回指定数量的候选歌曲
- 支持让 AI 选择候选歌曲后继续发送
- 支持多种发送方式：语音、远程音频链接、文件、音乐卡片
- 支持 `audio-url-model` 模式把音频链接返回给模型

## 安装

在 Koishi 插件市场中搜索并安装：

```text
chatluna-music-voice
```

本插件需要同时启用 ChatLuna

## 基础配置

- `工具名称`：注册到 ChatLuna 的工具名称，默认是 `music_voice`
- `工具描述`：显示在 ChatLuna 工具列表里的描述
- `每个平台每次搜索返回的候选歌曲数量`：网易云音乐和 QQ 音乐各返回 N 条
- `启用网易云音乐搜索`：默认开启
- `启用 QQ 音乐搜索`：默认关闭，QQ 音乐接口和卡片发送并不稳定

## 发送方式

- `下载音频后发送语音`：下载音频后通过当前聊天发送语音
- `返回远程音频链接给模型`：不直接发送消息，把音频链接返回给模型处理
- `直接发送远程音频链接`：把远程音频 URL 作为文本发送
- `把远程音频链接作为文件发送`：把远程音频 URL 作为文件发送
- `按歌曲来源发送音乐卡片`：网易云音乐发送网易云卡片，QQ 音乐发送 QQ 音乐卡片

默认情况下，AI 不能临时切换发送方式，只会使用 Koishi 前端中配置好的默认发送方式。如果需要让 AI 根据用户要求切换发送方式，可以开启“允许 AI 临时切换发送方式”

## 本地 302 短链

`audio-url-model` 模式返回的远程音频链接可能很长。开启本地 302 短链后，插件会把返回给模型的链接替换成 Koishi 本地短链，请求短链时再 302 跳转到原始音频 URL

这个功能默认关闭，使用时需要注意：

- 只影响“返回远程音频链接给模型”模式
- Koishi 不会下载音频，也不会代理音频流
- OneBot 必须能访问 Koishi 的短链地址

如果 OneBot 和 Koishi 在同一局域网内，可以把短链访问基础地址设置为 OneBot 能访问的 Koishi 地址，例如：

```text
http://192.168.1.10:5140
```

## 注意事项

- QQ 音乐搜索和卡片发送并不稳定，可能因为歌曲、地区或平台限制失败
- 如果歌曲直链 API 不可用，候选歌曲可能能搜索到，但链接获取会失败
